## L1 — Rule Signal
- **rule_type**: guidance
- **issuing_body**: Cloud Security Alliance (CSA)
- **document_id**: CSA Enabling Zero Trust for Cellular Networks（新出版品）
- **status**: final

## L2 — Responsibility Structure
- **affected_roles**: 電信安全工程師、雲端原生 5G 架構師、網路安全工程師、CISO
- **shift_type**: new
- **shift_summary**: CSA 針對電信環境發布零信任實施指引，提供從範疇龐大的蜂巢式網路（含 RAN、5G core、OSS/BSS、漫遊互聯）出發的優先化實施起點，解決電信業零信任落地的入門挑戰。

## L3 — Risk Domains
- 零信任架構（電信環境）
- 5G 核心網路安全
- 電信 API 暴露面管理

## L4 — Obligation & Evidence
- **new_obligations**:
  - 依 CSA 電信零信任指引識別並優先化實施起點
  - 覆蓋使用者設備、分散式 RAN、傳輸、雲原生 5G core、OSS/BSS 及虛擬化基礎設施
  - 處理漫遊互聯、Exposure API 及合作夥伴生態系的零信任需求
- **evidence_requirements**:
  - 電信環境零信任成熟度評估
  - 各層面（RAN/Core/BSS）零信任控制實施計畫
- **enforcement_signal**: recommended
