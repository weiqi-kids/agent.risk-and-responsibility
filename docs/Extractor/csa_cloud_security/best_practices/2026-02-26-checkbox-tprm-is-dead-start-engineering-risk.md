## L1 — Rule Signal

## L2 — Responsibility Structure
- **affected_roles**: TPRM 分析師、風險管理團隊、合規主管、安全工程師、CISO
- **shift_type**: expanded
- **shift_summary**: 傳統勾選式第三方風險管理（TPRM）無法應對現代供應商數量爆增的挑戰，組織應轉向「風險工程」方法——分析實際供應商連接、執行針對性技術測試並採取具體行動。

## L3 — Risk Domains
- 第三方與供應鏈風險
- 存取控制與權限管理
- 合規框架有效性

## L4 — Obligation & Evidence
- **new_obligations**:
  - 在評估前先盤點並分析供應商與關鍵系統的實際技術連接
  - 以對抗性思維（攻擊者視角）評估供應商整合風險
  - 執行針對性漏洞測試取代廣泛問卷
  - 依發現結果立即採取修復行動（撤銷權限、修改合約條款、換供應商）
- **evidence_requirements**:
  - 供應商依賴關係圖（含技術連接細節）
  - 針對性漏洞測試報告
  - 修復行動記錄（時間戳、執行人員、結果）
- **enforcement_signal**: recommended
