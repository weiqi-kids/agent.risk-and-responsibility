## L1 — Rule Signal
- **rule_type**: revision
- **issuing_body**: Cloud Security Alliance (CSA)
- **document_id**: CCM v4.1 / CAIQ v4.1
- **status**: final

## L2 — Responsibility Structure
- **affected_roles**: Cloud service providers, STAR Registry participants, compliance teams, cloud security auditors, security architects
- **shift_type**: expanded
- **shift_summary**: Organizations currently certified under CCM v4.0.x must transition to CCM v4.1 by December 2027, with the new version expanding controls across five critical domains and introducing enhanced interoperability with leading standards and regulatory frameworks.

## L3 — Risk Domains
- Cloud infrastructure security (Datacenter Security — DCS domain)
- Logging and monitoring gaps (LOG domain)
- Security incident management obligations (SEF domain)
- Supply chain risk management (STA domain)
- Threat and vulnerability management (TVM domain)
- Regulatory alignment and interoperability with external frameworks

## L4 — Obligation & Evidence
- **new_obligations**:
  - Adopt CCM v4.1 for all new STAR Registry submissions (required after December 2027)
  - Migrate existing STAR Registry certifications from v4.0.x to v4.1 within the two-year transition window (by December 2027)
  - Implement 11 new control specifications across DCS, LOG, SEF, STA, and TVM domains
  - Remove deprecated IAM control removed in v4.1
  - Respond to 283 aligned CAIQ v4.1 questions for conformance assessment
  - After January 2028, v4.0.x submissions will no longer be accepted
- **evidence_requirements**:
  - STAR Registry submission updated to CCM v4.1 framework
  - Completed CAIQ v4.1 (283 questions) demonstrating control alignment
  - Documentation of new controls implemented across DCS, LOG, SEF, STA, TVM domains
  - Mapping documentation showing alignment with external standards and regulatory frameworks
- **enforcement_signal**: mandatory
