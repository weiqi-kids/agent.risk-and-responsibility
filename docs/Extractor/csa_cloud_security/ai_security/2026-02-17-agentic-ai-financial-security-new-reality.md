## L2 — Responsibility Structure
- **affected_roles**: CISOs, security architects, financial services security teams, healthcare security teams, cloud-native engineering teams, compliance officers
- **shift_type**: expanded
- **shift_summary**: Agentic AI systems operating in production environments — automating workflows, moving data, calling APIs, and making decisions at machine speed — require organizations in financial services, healthcare, and cloud-native engineering to fundamentally redefine their security models beyond traditional user-centric controls.

## L3 — Risk Domains
- Agentic AI operating autonomously in production financial and healthcare environments
- API-level automation creating new data movement and access control risks
- Machine-speed decision making outpacing human oversight capabilities
- Security model gaps between human-centric identity frameworks and non-human AI agents
- Cross-sector AI security risks in financial services, healthcare, and cloud engineering

## L4 — Obligation & Evidence
- **new_obligations**:
  - Assess whether existing security models account for non-human AI agents operating continuously in production
  - Extend access control and monitoring frameworks to cover AI agent API calls, data movements, and automated decisions
  - Implement security controls that operate at machine speed, not human review cycles
  - Define security boundaries for agentic AI systems in regulated industries (financial services, healthcare)
  - Establish governance frameworks distinguishing between agentic AI actions and human-authorized actions
- **evidence_requirements**:
  - Security architecture documentation covering agentic AI system boundaries and controls
  - Access control policies extended to non-human AI agents
  - Audit logs capturing AI agent decisions, API calls, and data movements
  - Evidence that security model has been assessed and updated for agentic AI deployment
- **enforcement_signal**: recommended
