# L3：實務衝擊評估

**操作面衝擊**：
- 當前多數企業缺乏正式自主性分級
- Level 4（高度自主）需要 24/7 持續監控及即時終止機制，並要求高層主管簽核
- Level 5（完全自主）明確指出當前不宜企業部署

**財務/資源衝擊**：
- 控制措施應隨自主性級別升高而加強
- 高等級自主性需要更多監控與治理投資

**治理衝擊**：
- 需建立分類系統
- 技術強制執行邊界，不只偵測違規，而是從技術層面阻止越界
- 完整日誌記錄，所有自主決策須可追溯與歸責

# L4：落地執行指引

**六個自主性級別**：

| 級別 | 名稱 | 核心特徵 | 風險等級 |
|------|------|----------|----------|
| **Level 0** | 無自主性 | AI 僅提供資訊，人類執行所有行動 | 最低 |
| **Level 1** | 輔助型 | AI 可執行，但每個動作需人類明確批准 | 低 |
| **Level 2** | 監督型 | 人類批准計劃後，AI 自主執行批次任務 | 中低 |
| **Level 3** | 條件型 | AI 在定義邊界內自主決策，超出時才上報 | 中（取決於邊界定義） |
| **Level 4** | 高度自主 | 最低限度監督，人類轉為異常監控角色 | 高 |
| **Level 5** | 完全自主 | 自我設定目標，**當前不宜企業部署** | 極高 |

**風險差異**：
- **Level 0-1**：風險最低，控制重點在輸出品質
- **Level 3**：風險取決於邊界定義的嚴謹程度
- **Level 4**：需要「24/7 持續監控」及「即時終止機制」，並要求高層主管簽核

**治理建議**：

1. **建立分類系統**
   - 目前多數企業缺乏正式自主性分級
   - 需為每個 AI 代理明確標示自主性級別

2. **技術強制執行邊界**
   - 不只偵測違規
   - 從技術層面阻止越界

3. **比例性治理**
   - 控制措施應隨自主性級別升高而加強
   - 避免過度治理低風險系統

4. **完整日誌記錄**
   - 所有自主決策須可追溯與歸責
   - 建立審計軌跡

**框架靈感來源**：
- 汽車業 SAE J3016 標準（自動駕駛分級）
- 目標是建立「共同詞彙」供監管者、企業與保險業溝通使用

**Level 5 警告**：
- 文章明確指出完全自主（自我設定目標）當前不宜企業部署
- 風險過高，控制機制尚未成熟
