## L1 — Rule Signal
- **rule_type**: final
- **issuing_body**: NIST
- **document_id**: NIST IR 8349
- **status**: final

## L2 — Responsibility Structure
- **affected_roles**: IoT device manufacturers, developers, network operators, cloud providers, researchers, network administrators
- **shift_type**: new
- **shift_summary**: IoT 裝置製造商與網路管理者需採用標準化方法來描述和記錄裝置網路行為,以支援 MUD 規範並實施適當的網路存取控制

## L3 — Risk Domains
- IoT security
- Network segmentation
- Device identification
- Access control
- Supply chain transparency

## L4 — Obligation & Evidence
- **new_obligations**: IoT 利害關係人應使用本方法論捕獲、記錄並描述 IoT 裝置在各種使用情境與條件下的完整網路行為範圍;建議使用 MUD-PD 開源工具自動化裝置描述與 MUD 檔案生成;網路管理者需識別每個裝置並確定其必要的網路通訊需求以實施適當的存取控制
- **evidence_requirements**: MUD (Manufacturer Usage Description) 檔案,記錄裝置運作所需的網路通訊;裝置網路行為描述文件,涵蓋多種使用情境;網路存取控制政策文件,基於裝置身份與通訊需求
- **enforcement_signal**: recommended
