## L1 — Rule Signal
- **rule_type**: revision
- **issuing_body**: NIST
- **document_id**: SP 800-218r1
- **status**: public_comment

## L2 — Responsibility Structure
- **affected_roles**: Software developers, software development organizations, software acquirers, procurement teams, supply chain security managers
- **shift_type**: expanded
- **shift_summary**: 擴充安全軟體開發框架至 1.2 版,新增改進的實務作法、任務與範例,協助組織減少已發布軟體的漏洞數量並建立採購方與供應商間的共同語彙

## L3 — Risk Domains
- Software vulnerabilities in released products
- Exploitation impact mitigation
- Root cause vulnerability analysis
- Software supply chain security
- Vendor-acquirer communication gaps
- Vulnerability recurrence prevention

## L4 — Obligation & Evidence
- **new_obligations**: Implement new and improved practices, tasks, and examples from SSDF v1.2; reduce number of vulnerabilities in released software; mitigate potential impacts of exploitation; address root causes of vulnerabilities; establish common vocabulary for secure software development in supplier communications per Executive Order 14306
- **evidence_requirements**: Vulnerability reduction metrics; secure development practice documentation; root cause analysis reports; supplier communication records demonstrating SSDF vocabulary usage
- **enforcement_signal**: mandatory

## Notes
Public comment period open through January 30, 2026. This is Revision 1 of SP 800-218, prompted by Executive Order 14306. Initial public draft phase.
