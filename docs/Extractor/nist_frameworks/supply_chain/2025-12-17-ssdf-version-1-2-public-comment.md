## L1 — Rule Signal
- **rule_type**: revision
- **issuing_body**: NIST
- **document_id**: SP 800-218r1 (Revision 1)
- **status**: public_comment

## L2 — Responsibility Structure
- **affected_roles**: Software producers, software developers, software acquirers, federal agencies, software supply chain vendors
- **shift_type**: expanded
- **shift_summary**: SSDF Version 1.2 introduces new and improved practices, tasks, and examples for secure software development, expanding guidance for software producers and acquirers to mitigate vulnerability risks throughout the software development lifecycle

## L3 — Risk Domains
- Software supply chain security
- Software vulnerability management
- Secure software development lifecycle (SDLC)
- Vulnerability root cause prevention
- Exploitation impact mitigation

## L4 — Obligation & Evidence
- **new_obligations**:
  - Implement new and improved secure software development practices per SSDF Version 1.2
  - Integrate secure practices into each SDLC model used by the organization
  - Communicate security requirements through software acquisition processes
  - Address recommendations for mitigating software vulnerability risks per Executive Order 14306
- **evidence_requirements**:
  - Documentation of SSDF practices integration into SDLC
  - Records of software acquisition security requirements
  - Vulnerability mitigation measures implemented
- **enforcement_signal**: recommended

## Notes
Public comment period open through January 30, 2026. This revision builds upon previous SSDF versions with enhanced practices and examples.
