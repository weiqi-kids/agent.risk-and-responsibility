# ISO/IEC TS 27103:2026 — Cybersecurity — Guidance on using ISO and IEC standards in a cybersecurity framework

## L1: 基本資訊（必填）

**title**: Cybersecurity — Guidance on using ISO and IEC standards in a cybersecurity framework
**iso_refno**: ISO/IEC TS 27103:2026
**edition**: 1
**date**: 2026-02-06
**ics_code**: 35.030
**technical_committee**: ISO/IEC JTC 1/SC 27
**source_layer**: iso_standards
**original_content**: This document provides guidance on how to leverage existing ISO and IEC standards in a cybersecurity framework.

## L2: 責任轉移判定（必填）

**shift_type**: new
**enforcement_signal**: informational
**rule_type**: Published
**affected_party**: Cybersecurity framework implementers, risk managers, compliance officers
**explanation**: 此為新發布的技術規範（TS），提供如何在網路安全框架中運用現有 ISO 和 IEC 標準的指引。作為技術規範，本質上為資訊性質，幫助組織將 ISO/IEC 標準體系整合到諸如 NIST CSF、MITRE ATT&CK 等網路安全框架中。

## L3: 歷史脈絡（選填）

**replaces**: ISO/IEC TS 27110:2021 (standard/72437)
**context**: 網路安全框架（如 NIST Cybersecurity Framework、CIS Controls）廣泛應用於產業實務，但與 ISO/IEC 標準體系的對接往往不清晰。本技術規範提供了橋樑，幫助組織理解如何將 ISO/IEC 27000 系列等標準融入框架驅動的網路安全計畫。

## L4: 連動影響（選填）

**related_shifts**:
- ISO/IEC 27001（資訊安全管理系統）— 核心標準應用
- ISO/IEC 27002（資訊安全控制措施）— 控制措施對應
- NIST Cybersecurity Framework — 框架整合指引
- CIS Controls — 產業框架協調
- ISO/IEC 27005（資訊安全風險管理）— 風險管理對應

**cross_domain_impact**: 對金融、關鍵基礎設施、政府機關等使用網路安全框架的組織都有影響。促進標準化方法與框架驅動方法的整合，降低組織實施多重要求的複雜度。
