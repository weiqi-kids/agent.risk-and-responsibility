# Corrigendum to Council Implementing Regulation (EU) 2025/1980 concerning restrictive measures against Iran

- **CELEX Number**: 32025R1980R(01)
- **Document Type**: corrigendum
- **Binding Force**: directly_applicable

- **Implementing Act of**: Regulation (EU) No 267/2012

## Legal Basis
Implements Regulation (EU) No 267/2012 concerning restrictive measures against Iran.

## Objective
Technical correction of SWIFT/BIC code formatting errors in the annexes listing Iranian financial institutions subject to EU restrictive measures.

## Key Provisions

### Specific Corrections
1. **Entry 156 (Annex, Page 29)**: Corrects SWIFT code from "SCERIRTH KSH" to "SCERIRTHKSH" for the Kish Island branch
2. **Entry 173 (Annex, Page 33)**: Amends SWIFT identifier from "REF AIRTH" to "REFAIRTH"

## Risk & Responsibility Impact

### Shift Type
- clarified

### Responsibility Changes
Technical clarification of financial institution identification codes ensures accurate implementation of sanctions compliance obligations by financial institutions and competent authorities.

### Enforcement Signal
- mandatory

### Sectors Affected
- Financial services
- Compliance and sanctions enforcement
- International trade

### Jurisdictional Applicability
**Member States Bound**: All EU Member States
**EEA Relevance**: Yes (restrictive measures typically extend to EEA)
**Third Country Effect**: Extraterritorial application to entities engaged in transactions with listed Iranian entities

## Key Obligations
1. Financial institutions must use corrected SWIFT/BIC codes when identifying sanctioned Iranian entities
2. Compliance officers must update internal sanctions screening databases
3. Competent authorities must apply corrected identifiers in enforcement actions

## Implementation Requirements
Immediate application from date of publication. No transposition required (directly applicable regulation).
